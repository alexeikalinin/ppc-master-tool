# Подключение Яндекс API: Wordstat + Direct

Полная инструкция по получению доступа к Яндекс.Вордстат и Яндекс.Директ API.

---

## Что нужно в итоге

В `.env` файле проекта должны быть:
```
YANDEX_CLIENT_ID=...
YANDEX_CLIENT_SECRET=...
YANDEX_WORDSTAT_TOKEN=...
YANDEX_DIRECT_TOKEN=...
YANDEX_REFRESH_TOKEN=...
```

---

## Шаг 1. Создать OAuth-приложение

1. Открой **https://oauth.yandex.ru**
2. Нажми **«Зарегистрировать новое приложение»**
3. Заполни:
   - **Название:** PPC Master Tool (или любое своё)
   - **Платформа:** Веб-сервисы
   - **Callback URI:** `https://oauth.yandex.ru/verification_code`
4. В разделе **«Доступы»** найди и включи:
   - **Яндекс.Директ** → «Управление кампаниями» (`direct:api`)
   - **Яндекс.Вордстат** → доступ к Wordstat (если есть отдельный пункт)
5. Сохрани приложение
6. Скопируй **ClientID** и **Client secret** → вставь в `.env`:
   ```
   YANDEX_CLIENT_ID=твой_client_id
   YANDEX_CLIENT_SECRET=твой_client_secret
   ```

> ⚠️ Важно: scope `direct:api` должен быть включён именно на этом шаге, в настройках приложения. Если добавить его позже — старый токен не обновится автоматически, нужно будет получать новый.

---

## Шаг 2. Подать заявку на Wordstat API

1. Открой **https://wordstat.yandex.ru/api** (или зайди через Яндекс.Директ → Инструменты → API → «Мои заявки»)
2. Нажми **«Подать заявку»**
3. Заполни форму:
   - Тип: Прямой рекламодатель / Разработчик
   - Язык: Python
   - Протокол: JSON
   - Цель: анализ ключевых слов для автоматизации PPC
4. После одобрения (обычно 1–3 дня) Wordstat API будет доступен

---

## Шаг 3. Подать заявку на Direct API

1. Открой **https://direct.yandex.ru** → левое меню → **«Инструменты»** → **«API»**
   Или напрямую: **https://direct.yandex.ru/registered/main.pl?cmd=apiSettings**
2. Перейди на вкладку **«Мои заявки»**
3. Нажми **«Новая заявка»**
4. Заполни форму:
   - **Специфика работы:** «Прямой заказчик и автоматизируете управление собственными кампаниями»
   - **Язык:** Python
   - **Протокол:** JSON
   - **Версия библиотек:** httpx 0.27 (или твоя версия)
   - **Логины в Директе:** твой логин (например, alexeikalinin1)
   - **Для чего приложение:** отметь «Автоматизировать регулярную работу», «Получение статистики», «Подбор ключевых слов»
   - **Основные функции:** отметь «Управление кампаниями и объявлениями» + «Получение статистики и отчётов» + «Подбор ключевых слов»
5. Отправь заявку → ожидай одобрения (обычно 1–5 дней)
6. Статус можно проверить там же, на вкладке «Мои заявки»

---

## Шаг 4. Получить токен

После одобрения заявки запусти скрипт получения токена:

```bash
cd "твой_проект"
source .venv/bin/activate
python scripts/get_yandex_token.py
```

Скрипт:
1. Откроет браузер с URL авторизации (scope=direct:api, force_confirm=yes)
2. Войди в Яндекс под нужным логином
3. Нажми **«Разрешить»**
4. Тебя перенаправит на страницу с кодом вида `https://oauth.yandex.ru/verification_code?code=XXXXXXX`
5. Скопируй значение параметра `code=...` из URL и вставь в терминал
6. Скрипт выведет токены

Добавь в `.env`:
```
YANDEX_DIRECT_TOKEN=полученный_токен
YANDEX_WORDSTAT_TOKEN=полученный_токен   # тот же токен
YANDEX_REFRESH_TOKEN=полученный_refresh_token
```

> ⚠️ Если браузер не открылся — скопируй URL из вывода скрипта и открой вручную.

> ⚠️ Токен живёт ~1 год. Для обновления без повторного входа используй refresh_token (скрипт `scripts/refresh_token.py`).

## Шаг 5. Проверить что всё работает

```bash
source .venv/bin/activate && python scripts/test_direct_api.py
```

Должно вывести:
```
✅ OK  campaigns
✅ OK  forecast
✅ OK  reports
```

## Возможные ошибки

### Ошибка 58 — «Незавершённая регистрация»

**Причины:**
- Заявка на Direct API ещё не одобрена
- Токен получен без scope `direct:api` (приложение не имело этого доступа на момент авторизации)
- Аккаунт с нероссийской валютой (BYN, KZT) — может требовать ручной активации через поддержку

**Решение:**
1. Убедись что в OAuth-приложении включён доступ `direct:api` (Шаг 1)
2. Убедись что заявка одобрена (Шаг 3)
3. Если всё одобрено но ошибка есть — напиши в поддержку (см. ниже)

### Ошибка 401 — Unauthorized

Токен невалидный или истёк. Повтори Шаг 4.

### Ошибка 404 на Forecast API

Следствие ошибки 58 — исчезнет после её решения.

## Написать в поддержку Яндекс Директ

Если ошибка 58 не уходит при активном аккаунте:

1. Открой **https://yandex.ru/support/direct/troubleshooting/api.html**
2. Заполни форму:
   - **Сообщение:** «При вызове API v5 все методы возвращают ошибку 58 «Незавершённая регистрация». Аккаунт активен в веб-интерфейсе, заявка одобрена [дата]. Просьба активировать доступ к API.»
   - **Логин:** твой логин
   - **RequestId:** из вывода тест-скрипта
   - **Код запроса:** `{"method":"get","params":{"SelectionCriteria":{},"FieldNames":["Id","Name","Status"]}}`
   - **Код ответа:** `{"error":{"error_code":58,"error_string":"Незавершенная регистрация"}}`
